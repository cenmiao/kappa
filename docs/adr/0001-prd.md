# PRD：我想静静的刷题

## Problem Statement

用户拥有一套包含 4000+ 道题目的题库（两个 docx 文件，一个有答案和解析、一个没有），需要在手机上随时进行练习。用户希望：练习时优先呈现无答案的题目，做完一套卷子后统一查看成绩和错题解析，针对错题自动强化练习。此外，用户有强烈的离线使用需求——手机安装应用后，即使 PC 关机也能独立刷题。

## Solution

一款名为"我想静静的刷题"的 PWA 应用，部署在 GitHub Pages 上。手机首次访问安装后，全部资源缓存至本地，之后完全离线可用。

核心循环：打开应用 → 随机抽取 80 题（含错题强化） → 单题沉浸式作答（支持不确定标记） → 交卷 → 查看成绩报告（编号答题卡 + 错题解析 + 需复习列表 + 针对重练） → 查看历史趋势。

## User Stories

1. 作为刷题用户，我希望每次打开应用能快速开始随机练习，无需复杂设置
2. 作为刷题用户，我希望随机练习固定为 80 道题（单选 40 + 多选 20 + 判断 20），满分 100 分
3. 作为刷题用户，我希望每次随机练习自动融入 10 道历史错题，以强化薄弱环节
4. 作为刷题用户，我希望题目一屏只显示一道，让我专注作答不被干扰
5. 作为刷题用户，我希望在答题时能回头修改之前的答案
6. 作为刷题用户，我希望对拿不准的题目做"不确定"标记，即使选对了交卷后也能看到提醒
7. 作为刷题用户，我希望做题期间看不到答案，交卷后才统一展示
8. 作为刷题用户，我希望交卷前有确认弹窗，防止误触，并提醒未答题数量
9. 作为刷题用户，我希望交卷后看到完整的成绩报告：总分、正确率、各题型得分
10. 作为刷题用户，我希望答题卡用编号 + 底色显示每道题的结果（绿 = 正确、红 = 错误、黄 = 蒙对），一目了然
11. 作为刷题用户，我希望点击答题卡中任意题号能展开该题的完整内容、正确答案和解析
12. 作为刷题用户，我希望成绩报告底部汇总所有错题和不确定题到一起，形成"需复习列表"
13. 作为刷题用户，我希望成绩报告中有"针对需复习的题目重新练习"按钮，方便查漏补缺
14. 作为刷题用户，希望全部答对时看到庆祝反馈，没有需要复习的题目
15. 作为刷题用户，我希望按题库编号顺序练习，支持续上次的进度
16. 作为刷题用户，希望查看历史练习记录，了解自己的进步趋势
17. 作为刷题用户，希望查看每次练习的得分曲线、各题型正确率变化
18. 作为刷题用户，希望历史记录为空时有友好的引导提示
19. 作为刷题用户，希望应用安装到手机主屏幕后完全离线可用
20. 作为刷题用户，希望首次访问时有 PWA 安装引导提示
21. 作为刷题用户，希望在答题中误关页面时浏览器弹出确认，防止进度丢失
22. 作为刷题用户，希望在题库加载失败时看到错误提示和重试按钮

## Implementation Decisions

### 技术栈
- React + Vite + TypeScript
- TailwindCSS v4 用于样式
- vite-plugin-pwa 提供 PWA 能力
- IndexedDB 用于本地数据持久化
- GitHub Pages 用于部署

### 题库处理
- 源数据：两个 docx 文件（tiku-with-answer.docx 和 tiku-without-answer.docx）
- 使用 pandoc 将 docx 转为纯文本，正则解析提取结构化数据
- 解析脚本分两步：先全量干跑生成格式异常报告，再正式输出 JSON
- 输出 `questions.json` 随应用部署，首次打开时导入 IndexedDB 的 questions 表
- 校对两套文件的题目编号和内容一致性

### 数据模型

Question 类型（来自原型 mock.ts）：

```typescript
type QuestionType = 'single' | 'multi' | 'tf'

interface Question {
  id: number
  type: QuestionType
  typeLabel: string        // "单选题" | "多选题" | "判断题"
  stem: string             // 题干
  options: string[]        // ["A. ...", "B. ...", ...]
  answer: string           // 单选: "A" | 多选: "A,B,C" | 判断: "A"/"B"
  explanation: string      // 解析
}
```

AnswerRecord 类型：

```typescript
interface AnswerRecord {
  questionId: number
  userAnswer: string
  isUncertain: boolean
}
```

Attempt 类型：

```typescript
interface Attempt {
  id: string
  date: string
  mode: 'random' | 'sequential'
  score: number
  total: number
  accuracy: number
  singleAccuracy: number
  multiAccuracy: number
  tfAccuracy: number
  answers: AnswerRecord[]
}
```

### IndexedDB 存储
- `questions` 表：只读，首次导入后不变
- `attempts` 表：不断追加，每条记录含完整作答详情

### 随机练习出题策略（错题强化）
- 总题数固定 80（单选 40、多选 20、判断 20）
- 从历史 attempts 中统计各题型错题池
- 目标填充 10 道错题，按题型 C 逻辑：错题按其题型占用对应名额
- 某题型错池不足 → 由其他题型错题补足至 10 道
- 总错题不足 10 → 余数由随机新题补齐
- 最终：各题型名额 = 错题占用 + 随机新题补齐

### 评分规则
- 单选题：1 分/题，共 40 分
- 多选题：2 分/题，必须完全匹配正确答案，共 40 分
- 判断题：1 分/题，共 20 分
- 满分 100 分

### 页面路由与组件
- `/` → 首页（HomePage）：三个入口卡片 + PWA 安装引导条
- `/quiz` → 答题页（QuizPage）：加载/失败/就绪三态 + beforeunload 拦截
- `/quiz?loadError=true` → 演示加载失败流程
- `/history` → 历史记录（HistoryPage）：统计概览 + 趋势图 + 可展开列表
- 交卷后内嵌展示 ResultPage（非独立路由）

### 答题页交互
- 单题沉浸模式，顶部细渐变色进度条
- 题型标签颜色：单选 = blue、多选 = purple、判断 = amber
- 右下角不确定标记按钮 ⚐/⚑ 切换
- 底部缩略图网格导航（已答 = 绿、当前 = indigo 环、未答 = 灰、标记 = ⚑）
- 非末尾题：`[←] [下一题] [交卷(小按钮)]`
- 末尾题：`[←] [✓ 交卷 (N/80 已答)] [→(禁用)]`
- 交卷确认弹窗（底部 sheet 样式，遮罩可关闭，未答题数红色高亮）

### 成绩报告
- 头部：indigo 渐变背景，总分大数字 + 白色正确率进度条
- 三列题型得分：单选/多选/判断各一列 "正确数/总数" + 微型进度条
- 答题卡：8 列网格，每格为题号数字，绿/红/黄底色区分结果
- 点击题号展开解析弹窗：完整题目 + 选项标记（← 你的选择、✓ 正确答案）+ 解析
- 需复习列表（错题 + 不确定题合并），空时显示 🎉 庆祝
- 底部三重按钮：琥珀色"针对 N 题重练"（有复习题时）+ 返回首页 + 重新练习

### 历史记录
- 空状态：📝 + 引导文案 + 开始练习按钮
- 有数据时：统计概览四格卡片 + SVG 折线趋势图 + 可展开练习列表
- 练习行点击展开：各题型正确率进度条 + 编号答题卡网格

### PWA 配置
- `viewport-fit=cover` 元标签
- Service Worker 预缓存全部静态资源 + 题库 JSON
- 应用图标："静"字 + indigo 背景
- 首页顶部 PWA 安装引导条（可 ✕ 关闭）

### 边缘/错误状态
- 题库加载中：居中 spinner + "正在加载题库..."
- 题库加载失败：😵 emoji + 错误提示 + 重试按钮 + 返回首页按钮
- 历史为空：友好的空状态引导
- 成绩全对：🎉 庆祝文案，不显示需复习列表标题
- 答题中刷新/关闭：beforeunload 浏览器确认框

## Testing Decisions

### 测试原则
- 只测试外部行为，不测试实现细节
- 优先集成测试，覆盖完整用户流程
- 对出题算法（错题强化逻辑）进行单元测试

### 测试模块

**1. 题库解析脚本**
- 正常格式解析正确性
- 边界情况：编号不连续、选项数量异常、缺失答案/解析、题干含换行
- 两套文件交叉验证

**2. 出题算法**
- 固定 80 题约束不变
- 各题型错题充足时正确按名额分配
- 某题型错题不足时跨题型补足
- 总错题不足 10 道时随机补齐
- 无历史错题时全部随机

**3. 评分逻辑**
- 单选/判断：用户答案与正确答案完全匹配
- 多选：完全匹配才得分，漏选/多选均不得分
- 未作答计为错误

**4. IndexedDB**
- 首次导入 questions 表
- attempts 记录读写
- 错题池统计准确性

**5. 用户流程（集成）**
- 首页 → 开始练习 → 答完全部题目 → 交卷确认 → 成绩报告
- 答题中标记不确定题 → 交卷后归入需复习列表
- 历史记录为空时展示空状态
- 历史记录有数据时展开查看详情

### 原型参考
- 原型中 `useQuizState.ts` 的状态接口可作为生产实现的起点
- 原型 `mock.ts` 中的类型定义可用于生产 schema

## Out of Scope

- 多题库支持（仅支持单套内置题库）
- 用户手动导入题库（不做自定义导入功能）
- 用户登录/注册（单用户单设备场景）
- 云端数据同步（纯本地存储）
- 社交分享功能
- 应用商店发布（仅 GitHub Pages + PWA）
