# 原型设计笔记

## 问题
"我想静静的刷题" PWA 应用的界面应该长什么样？

## 创建日期
2026-06-23

## 决定（已确认）
- **首页**: A-卡片网格 — 渐变大卡片主力入口 + 白色辅助卡片，视觉层次分明
- **答题页**: B-滑动沉浸 — 全屏沉浸式卡片 + 大号点击区域 + 底部缩略图网格导航
- **成绩报告页**: A-滚动报告 — 垂直长页面，所有信息依次排列

## 已清理
- 首页 B（信息列表）、C（单一焦点）
- 答题页 A（经典卡片）、C（纵向全览）
- 成绩报告 B（标签页模式）
- PrototypeSwitcher 浮动切换器组件

## 评审改进 (2026-06-23)

基于 `kappa-prototype-review.md` 完成：

### P0 修复
1. **交卷按钮全局可见** — 非最后一题时在右下角显示"交卷"小按钮，最后一题时升级为绿色大按钮
2. **交卷确认弹窗** — 点击交卷弹出确认对话框，区分有未答题/全部已答两种情况，提供取消和确定按钮
3. **答题卡网格改用编号+底色** — 绿色底=正确，红色底=错误，黄色底=蒙对，色块内显示题号数字

### P1 补充
4. **针对重练按钮** — 需复习列表下方增加"针对这 N 题重新练习"按钮（琥珀色）
5. **历史记录可点击展开** — 每条练习记录可点击展开详情，显示各题型正确率进度条和答题卡网格

## 拷问改进 (2026-06-23)

基于 `kappa-grilling-updates.md` 完成：

1. **历史记录空状态** — 无记录时显示 emoji + "还没有练习记录" + 开始随机练习按钮
2. **PWA 安装引导条** — 首页顶部 indigo 横幅 "点击下方 ··· 添加到主屏幕"，带关闭按钮
3. **成绩报告全对状态** — 零复习题时显示 🎉 "全部掌握！没有需要复习的题目"
4. **题库加载失败状态** — 加载中 spinner + 失败时 😵 错误提示 + 重试/返回首页按钮
5. **答题中刷新拦截** — `beforeunload` 事件，答题未交卷时关闭/刷新页面弹出确认框
